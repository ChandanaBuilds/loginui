i want to start the git so give me basic git knowledge in simple words and simple commands Great choice. Git is one of the first tools every developer should learn. Think of it like a save history system for your code. It lets you save changes, go back to previous versions, and work with other developers without overwriting each other's work. What is Git? Git is a Version Control System. Imagine you're writing a project report. Instead of saving: Project.docx Project_final.docx Project_final2.docx Project_latest.docx Project_latest_final.docx Git automatically keeps all these versions for you. What is GitHub? Git stores code on your computer. GitHub stores your Git repository online so you can: Backup your code Share it with others Work in teams Deploy projects Think of it like: Git = Your notebook GitHub = Google Drive for your notebook Important Git Terms Repository (Repo) A project folder managed by Git. Example: LaptopStore/ This entire folder becomes a Git repository. Commit A commit is like taking a snapshot of your project. Example: Version 1 ↓ Version 2 ↓ Version 3 Each snapshot is called a commit. Branch A separate copy of your project. Example: main | feature-login | feature-payment You can develop new features without affecting the main project. Push Upload your code to GitHub. Computer ↓ GitHub Pull Download the latest code from GitHub. GitHub ↓ Computer Clone Copy a GitHub repository to your computer. Git Workflow Create Project ↓ Initialize Git ↓ Write Code ↓ Add Changes ↓ Commit ↓ Push to GitHub Basic Git Commands 1. Check Git Version git --version Example git version 2.50.1 2. Configure Your Name (Only Once) git config --global user.name "Your Name" Example git config --global user.name "Chandana" 3. Configure Your Email (Only Once) git config --global user.email "your@gmail.com" Example git config --global user.email "abc@gmail.com" 4. Initialize Git Inside your project: git init Output: Initialized empty Git repository Now your folder is tracked by Git. 5. Check Status git status Shows: New files Modified files Deleted files This is the command you'll use most often. 6. Add Files Add one file: git add index.html Add all files: git add . . means "everything in the current folder." 7. Commit Changes git commit -m "Initial project setup" Example: git commit -m "Added login page" Always write a meaningful message describing what changed. 8. View Commit History git log Short version: git log --oneline Example: a23d12 Added login page b42c33 Fixed navbar c21ab1 Initial commit 9. Connect to GitHub git remote add origin https://github.com/username/project.git This tells Git where the online repository is. 10. Push Code First push: git push -u origin main Later pushes: git push 11. Clone a Repository git clone https://github.com/username/project.git This downloads the project to your computer. 12. Download Latest Changes git pull 13. See Branches git branch Output: * main The * indicates your current branch. 14. Create a New Branch git branch feature-login 15. Switch to a Branch git checkout feature-login Or using the newer command: git switch feature-login 16. Create and Switch in One Step git checkout -b feature-login Or: git switch -c feature-login 17. Merge a Branch Go back to the main branch: git switch main Then merge: git merge feature-login 18. Remove a Branch git branch -d feature-login Daily Workflow Every day you'll usually do: git status git add . git commit -m "Describe your changes" git push That's the core Git workflow you'll use most often. Example You create a React project. Step 1 git init Step 2 git status Step 3 git add . Step 4 git commit -m "Created React project" Step 5 git remote add origin https://github.com/username/react-project.git Step 6 git push -u origin main Your project is now stored on GitHub. Step 4: Configure Git (One-Time Setup) Set your name: git config --global user.name "Your Name" Example: git config --global user.name "Chandana" Set your email (use the email associated with your GitHub account if you have one): git config --global user.email "your-email@gmail.com" Example: git config --global user.email "chandana@gmail.com" Verify your configuration: git config --list You should see entries similar to: user.name=Chandana user.email=chandana@gmail.com Step 5: Test Git Create a test folder: mkdir GitPractice Go into it: cd GitPractice Initialize Git: git init You should see a message similar to: Initialized empty Git repository in ... Check the status: git status If you see output like: On branch main No commits yet nothing to commit then Git is working correctly. If You Use VS Code Open VS Code and open the integrated terminal with: Ctrl + ` Then run: git --version VS Code should detect Git automatically if it is installed correctly. If git Is Not Recognized If you get an error like: 'git' is not recognized as an internal or external command it usually means: Git isn't installed, or Git wasn't added to your system's PATH during installation. In that case, reinstall Git and ensure you choose "Git from the command line and also from 3rd-party software" when prompted.
